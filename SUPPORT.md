# Support

Credential Airlock is open source and self-hosted. The fastest support path is a
clear, reproducible GitHub issue.

## Questions And Bugs

Use GitHub Issues:

<https://github.com/classeve-public/credential-airlock/issues>

Please include:

- Operating system and Node.js version.
- Credential Airlock version or commit.
- Exact command you ran.
- Expected behavior and actual behavior.
- Redacted logs or screenshots when useful.

Never include real API keys, sealed vault files, local CA private keys, admin
tokens, backups, or audit logs that may reveal your environment.

## Security Reports

Do not open a public issue for vulnerabilities. Follow [SECURITY.md](SECURITY.md)
and use GitHub Security Advisories:

<https://github.com/classeve-public/credential-airlock/security/advisories/new>

## Commercial Or Enterprise Use

Credential Airlock is Apache-2.0 open source. You can use, inspect, fork, and
ship it under that license. For enterprise hardening questions, threat-model
review, or third-party assessment coordination, open a GitHub discussion or
contact the maintainer listed in [SECURITY.md](SECURITY.md).
