# Code Of Conduct

Credential Airlock is a security tool. The project needs direct technical
critique, but it also needs a professional working environment.

## Expected Behavior

- Be precise, evidence-based, and respectful.
- Assume good faith while asking for reproducible proof.
- Keep security reports private until coordinated disclosure is complete.
- Redact secrets, tokens, vault material, and private infrastructure details.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Publishing another person's secrets, tokens, vault files, or private logs.
- Publicly disclosing an unpatched vulnerability before coordinated disclosure.
- Spam, social-engineering attempts, or issue flooding.

## Enforcement

Maintainers may edit, hide, lock, or delete comments and issues that put users
at risk or make the project unsafe to work in. Repeated or severe violations may
lead to being blocked from the repository.

For security-sensitive conduct issues, use the private vulnerability reporting
path in [SECURITY.md](SECURITY.md).
